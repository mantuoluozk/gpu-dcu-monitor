# 更新记录

## 2026-06-23

- CPU 信息新增具体型号/部件信息展示，优先读取 `dmidecode -t processor`，用于识别 7470 等更细型号；取不到时回退到系统通用型号。
- CPU 板块新增物理 CPU 翻页卡片，可逐颗查看 Socket、具体型号、部件号、核心/线程数和频率信息；负载显示改为标注 1m/5m/15m。
- CPU 翻页改为仅在每颗 CPU 明细确实不同时显示；同型号多路机器只展示汇总，避免重复页面。CPU 具体型号优先采用 `lscpu` 中带 OPN 的 `Model name`。
- DCU 卡详情新增 CU 数展示，优先从 `hy-smi -q` 解析，必要时尝试 `rocminfo` 作为兜底。
- 修正 `rocminfo` CU 解析：会跳过 Hygon CPU Agent，只将 `gfx*` / `BW*` / `C-3000` 等 DCU Agent 映射到 DCU 卡。
- 详情页视觉分区增加主题色标题条，CPU、内存、系统、DCU/GPU、模型镜像板块使用不同强调色，提升扫描辨识度。
- 修复详情页点击“刷新当前”后白屏的问题：单台刷新结果现在会正确合并回服务器列表，并保留已加载的模型/镜像详情。
- CPU 型号汇总改为按物理 CPU/socket 统计，不再把每个逻辑核心都算成一颗 CPU；详情页同时显示物理 CPU 数和逻辑核心数。
- 重构服务器详情页，将信息拆分为 CPU、内存、系统版本、DCU/GPU 卡状态、模型与镜像几个板块，减少系统和驱动信息挤在一起的问题。
- 新增单台服务器刷新入口：详情页可以只刷新当前机器的 DCU/GPU、CPU、内存、系统和驱动信息，不再必须触发全局刷新。
- 优化 CPU 与系统探针：CPU 型号支持多型号/多路汇总展示，驱动版本过滤会避开 `hy-smi` 横线和 banner 文本，并兼容纯版本号输出。
- CPU 功耗增加 `powercap`、`hwmon power*_input` 和 `sensors` 多层兜底；如果目标服务器没有暴露功耗传感器，界面仍会显示为空。

## 2026-06-22

- 新增 CPU 与系统信息采集：服务器刷新时会同步采集 CPU 利用率、核心数、CPU 型号、内存使用率、负载、运行时间、系统版本、内核架构和 GPU/DCU 驱动版本。
- 新增 CPU 温度和 CPU 功耗 best-effort 展示：优先读取 `sensors`、`/sys/class/thermal` 和 `powercap energy_uj`，无法读取时在界面显示为空，不影响 GPU/DCU 占用状态刷新。
- 前端服务器卡片新增 CPU、内存和温度摘要；服务器详情页新增“CPU 与系统信息”面板，集中展示系统版本、驱动版本、CPU 型号、功耗、温度和运行时间。
- 新增 `SYSTEM_SSH_TIMEOUT_MS` 环境变量控制系统探针超时，默认不超过 12 秒；系统探针失败不会把服务器判定为离线。

## 2026-06-15

- 将模型资产默认扫描深度从 12 层调整为 6 层，减少对深目录和慢挂载盘的无效扫描压力；仍可通过 `ASSET_SCAN_MAX_DEPTH` 按部署环境单独调大。
- 收窄模型资产默认扫描路径，移除 `/opt`、`/tpstor`、`/glusterfs-user-data`、`/Ring-2.5-1T` 等偏固定环境的目录；需要时可通过 `ASSET_PATHS` 单独配置。
- 重构前端看板视觉为更紧凑的资源调度台风格，将全局刷新、模型资产刷新和添加服务器移到顶部操作区，服务器卡片改为更清晰的卡位矩阵展示。
- 将前端升级为 React 组件化实现，并引入本地 React 运行时文件；页面改为“机柜调度室”风格，强化资源池总览、GPU/DCU 槽位灯阵、模型检索和右侧检查面板。
- 优化 React 看板首页的信息密度：移除常驻右侧详情列，点击服务器后再打开详情覆盖页；收窄左侧筛选栏、压缩服务器卡片高度，并将配色调整为更清爽的石墨蓝冷白风格。
- 修复服务器卡片底部标签在紧凑布局下被遮挡的问题，首页仅展示关键标签并用数量徽标汇总其余标签。
- 进一步简化首页服务器卡片，移除底部标签行并将编辑入口移到右上角，避免模型数、镜像数和盘点时间在紧凑布局下溢出卡片。
- 恢复首页服务器卡片的模型数、镜像数和盘点时间摘要，并放大卡片宽度与高度下限，确保资产摘要在卡片内完整显示。
- 隐藏顶部“刷新模型资产”手动入口，保留每日自动盘点和模型镜像检索能力，减少误触发深目录扫描的风险。
- 补充本地项目记忆中的双云端部署信息，记录昆山 / 天津与太原两个实例的访问地址、部署目录和部署注意事项；同时清理本地日志、旧部署包等运行产物。

## 2026-06-12

- 补充 Windows 本地调试说明：PowerShell 可能因执行策略拦截 `npm.ps1`，建议使用 `cmd.exe /c npm.cmd start`、`start-windows.bat` 或直接 `node server.js` 启动。
- 记录本机 Codex/Windows 环境访问差异：`localhost:3066` 可能因 IPv6 解析超时，遇到页面打不开时优先使用 `http://127.0.0.1:3066/`。
- 模型资产结果过滤框架和工程目录，例如 `vllm`、`LLaMA-Factory`、`stable-diffusion-webui`、`model-downloader-main`，避免把运行框架误当作模型展示。
- 修复模型路径详情滚动时偶尔跳动的问题，自动轮询刷新时会保留已加载的模型详情，并在重绘后恢复右侧详情和模型列表的滚动位置。

## 2026-06-11

- 保留“刷新模型资产”手动入口，方便本地测试和上线后立即盘点；同时模型路径盘点会每天 02:00 自动执行。
- 扩大模型资产扫描范围，默认覆盖 `/models`、`/model`、`/public`、`/data`、`/mnt`、`/home`、`/root`、`/workspace`、`/workspaces`、`/opt` 等常见目录，并识别更多权重、配置和推理引擎文件。
- 模型资产默认返回上限提升到 800 条，并支持通过 `ASSET_SCAN_MAX_DEPTH`、`ASSET_REFRESH_HOUR`、`ASSET_REFRESH_MINUTE` 调整扫描深度和自动盘点时间。
- 根据太原送测设备模型统计表补全 `/data1`、`/data2`、`/data_nvme*`、`/apps`、`/aidata`、`/storage`、`/tpstor`、`/glusterfs-user-data` 等模型落点，默认扫描深度提升到 12 层，条目上限提升到 1500 条。
- 收紧模型资产筛选：权重和推理引擎文件直接展示；`config.json`、`tokenizer.json` 等弱特征只用于确认模型目录，不再单独作为模型文件展示，减少普通项目目录混入。
- 单台服务器模型资产盘点默认超时从 30 秒提升到 120 秒，减少深目录扫描时被提前判定为盘点失败。
- 模型资产盘点改为按根目录独立限时扫描，单个大目录或慢挂载超时后会跳过该目录，尽量保留同一机器其他目录的盘点结果。
- 模型资产列表改为按模型目录聚合展示，不再把 `.gguf`、`.safetensors`、`.bin` 等权重文件逐条铺在界面上；这些文件名仍会参与搜索命中。
- 新增“更新日志”页面，用户可以在界面中直接查看每次功能变化和部署兼容性说明。
